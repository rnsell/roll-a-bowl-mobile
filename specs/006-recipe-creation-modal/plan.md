# Implementation Plan: Recipe Creation Modal

**Branch**: `006-recipe-creation-modal` | **Date**: 2026-03-18 | **Spec**: [spec.md](./spec.md)
**Input**: Feature specification from `/specs/006-recipe-creation-modal/spec.md`

## Summary

Add a recipe creation modal to the mobile app. Users tap the "+" button on the Recipes screen to open a full-screen modal with a scrollable form (name, ingredients, instructions). Ingredients are added via a bottom sheet with search. Family group members see an auto-share indicator. The backend handles family assignment automatically. Uses existing GraphQL mutations/queries from the web app backend.

## Technical Context

**Language/Version**: TypeScript 5.9 (strict mode)

**Testing**: Jest, React Native Testing Library
**Target Platform**: iOS and Android (Expo managed workflow)
**Project Type**: Mobile app
**Performance Goals**: 60fps animations, ingredient search results within 1 second
**Constraints**: Expo managed workflow, no native modules outside Expo SDK
**Scale/Scope**: Single modal screen with one bottom sheet sub-flow

## Design Decisions

### Modal Route

Add `app/create-recipe.tsx` as a modal route with `presentation: 'modal'` in the Stack navigator. This follows the existing pattern where `app/family-group.tsx` delegates to a screen component.

### Form Architecture

Single scrollable form (`ScrollBox`) with three sections:
1. **Recipe Name** — `TextInput` with validation
2. **Ingredients** — List of `IngredientRow` components with "Add Ingredient" button that opens the bottom sheet
3. **Instructions** — Multi-line `TextInput`

Header contains Cancel (left) and Create Recipe (right) buttons.

### Ingredient Bottom Sheet

`@gorhom/bottom-sheet` with two snap points:
- **Half-screen (50%)**: Search input + results list
- **Expanded (85%)**: After selecting an ingredient, expands to show quantity, measurement picker, preparation note, and optional toggle

The sheet dismisses after the user confirms adding the ingredient, returning to the main form.

### Family Group Indicator

Read `familyGroupName` from Redux user state (`state.user.profile?.familyGroupName`). If present, show a `Caption` below the recipe name input: "This recipe will be shared with [Family Name]". No additional API calls needed.

### Apollo Cache Refresh

After successful `CreateRecipe` mutation, use `refetchQueries: [GetRecipes]` to refresh the recipe list. When the modal dismisses, the recipes screen will already have the updated data.

### Validation

Client-side validation before mutation:
- Name: required, max 200 characters
- Ingredients: no validation (optional)
- Instructions: no validation (optional)

Show inline error below the name field if empty on submit.

### Discard Confirmation

Track form "dirty" state by comparing current values to initial empty state. If dirty and user attempts to dismiss (back gesture, Cancel button), show `Alert.alert` confirmation dialog.
